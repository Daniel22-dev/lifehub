Šifrovaný trezor dokumentů

Nahrát lokálně, mít po ruce v tomto zařízení

Sem patří soukromé PDF, obrázky, exporty, faktury nebo jiné podklady. Soubory se ukládají do IndexedDB v prohlížeči, nikoli do GitHub repozitáře. Nejde o šifrovaný archiv, ale o šifrovaný trezor daného zařízení.

Důležité: šifrovaný trezor není šifrovaný archiv. Při smazání dat prohlížeče, továrním resetu nebo přístupu jiné osoby k profilu prohlížeče může být obsah ztracený nebo dostupný. Citlivé zálohy ukládejte šifrovaně a mimo veřejný GitHub.

Celá aplikace

Soukromé exporty mohou obsahovat osobní finance, poznámky, URL a případně extrahovaný text z pásek. Originální PDF a soubory z šifrovaného trezoru se do JSON/MD exportu nepřibalují, zůstávají v IndexedDB.

Import přepíše současná lokální data až po potvrzení. PDF soubory uložené v IndexedDB zůstanou samostatně podle lokálního úložiště.

Notion

Balíček pro Notion

Pro vlastní soukromý Notion použijte Markdown pro dlouhé poznámky a CSV pro tabulkové databáze. Přímé API propojení vyžaduje bezpečný backend/proxy, ne veřejně uložený API token v HTML.

Anonymizace

Export pro veřejný GitHub nebo sdílení bez originálů

Anonymizovaný export neobsahuje PDF, soubory z šifrovaného trezoru, názvy souborů, URL, extrahovaný raw text, poznámkové obsahy ani jména zaměstnavatelů. U financí používá kvartály a finanční rozsahy, aby nebyly veřejně vidět přesné měsíce ani přesné částky.

Pozor: pokud je export veřejně vedle aplikace s vaším jménem v patičce, nejde o dokonalou anonymitu, ale o výrazné odstranění identifikátorů. Pro opravdu soukromá data používejte soukromou zálohu mimo veřejný repo.

Praktický workflow

1. AI vlákna

Po dobré odpovědi z ChatGPT/Claude uložte název, stručné shrnutí, tagy, použitelný prompt a další krok.

2. Výplatní pásky

Ve Financích načtěte PDF, zkontrolujte rozpoznané hodnoty a uložte. Čistá mzda se zapíše jako příjem daného měsíce; PDF může zůstat lokálně v archivu.

3. Šifrovaný trezor

Soukromé dokumenty nahrajte v části Archiv. Budou dostupné z aplikace na daném zařízení, ale nebudou součástí veřejného GitHubu.

4. Úkoly a nákupy

Rozdělujte podle naléhavosti. Dashboard pak ukáže, co je urgentní a kolik stojí plánované nákupy.

5. Zálohy a anonymizace

Soukromou JSON zálohu ukládejte mimo veřejný repozitář. Pro GitHub používejte anonymizovaný export bez PDF, názvů souborů a raw textu.

Omezení PDF importu

Aplikace čte textovou vrstvu PDF. Pokud zaměstnavatel posílá sken nebo obrázek v PDF, je nutné nejdříve použít OCR. PDF import čte jen prvních několik stran a má bezpečnostní limity. Rozpoznané částky jsou návrh; před uložením je zkontrolujte.
